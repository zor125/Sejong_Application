# UI

## 개요

이 문서는 간호학원 문제집 관리 시스템의 강사용 웹 화면 구조를 정의한다. 학생 앱 화면은 이 문서 범위에서 제외한다.

강사용 웹은 반복적인 관리 업무를 빠르게 처리하는 백오피스형 UI로 설계한다. 핵심 작업은 기수 관리, 학생 관리, 문제 관리, 문제집 제작 및 배포, 성적 분석이다.

## 프론트엔드 폴더 구조

React + Vite 기반으로 다음 구조를 사용한다.

```text
src/
  api/
    index.ts
  components/
    common/
      PageTitle.tsx
    layout/
      AdminLayout.tsx
  hooks/
    index.ts
  lib/
    routes.ts
  pages/
    auth/
      LoginPage.tsx
    admin/
      dashboard/
        index.tsx
      cohort/
        index.tsx
      student/
        index.tsx
      question/
        index.tsx
      workbook/
        index.tsx
      score/
        index.tsx
      settings/
        index.tsx
  types/
    index.ts
  utils/
    index.ts
  main.tsx
  router.tsx
  styles.css
```

### 폴더 역할

- `api`: REST API 호출 함수와 API 클라이언트 관리
- `components/common`: 여러 페이지에서 재사용되는 공통 UI 컴포넌트
- `components/layout`: 사이드바, 헤더, 본문 영역을 포함한 레이아웃 컴포넌트
- `hooks`: 공통 React hook
- `lib`: 라우트 상수, 설정값, 외부 라이브러리 래퍼
- `pages/auth`: 로그인 등 인증 화면
- `pages/admin`: 강사용 웹의 사이드바 메뉴별 페이지
- `types`: 공통 TypeScript 타입
- `utils`: 포맷터, 검증 함수 등 순수 유틸리티

## 라우팅 구조

라우팅은 `react-router-dom` 기준으로 구성한다.

| URL | 페이지 | 설명 |
| --- | --- | --- |
| `/` | Redirect | `/admin/dashboard`로 이동 |
| `/login` | `LoginPage` | 강사용 로그인 |
| `/admin` | `AdminLayout` | 강사용 웹 공통 레이아웃 |
| `/admin/dashboard` | `DashboardPage` | Dashboard |
| `/admin/cohorts` | `CohortPage` | 기수관리 |
| `/admin/students` | `StudentPage` | 학생관리 |
| `/admin/questions` | `QuestionPage` | 문제관리 |
| `/admin/workbooks` | `WorkbookPage` | 문제집관리 |
| `/admin/scores` | `ScorePage` | 성적분석 |
| `/admin/settings` | `SettingsPage` | 설정 |

### 라우팅 원칙

- `/admin` 하위 경로는 `AdminLayout`을 공통 부모로 사용한다.
- 사이드바 메뉴와 URL은 일대일로 대응한다.
- `/admin` 접근 시 기본 화면인 `/admin/dashboard`로 이동한다.
- 인증이 필요한 라우트 보호는 추후 `ProtectedRoute` 컴포넌트로 추가한다.
